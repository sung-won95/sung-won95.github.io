postgres replication 하는방법

    • MASTER 설정
    • STANDBY SERVER 세팅
    • replication 실행 확인

master 서버와 slave 서버에 각각 설정하는 방법이 다릅니다

MASTER 설정

  1. user 생성

    # psql을 Postgres 유저로 접속 후 유저 생성

sudo -u postgres psql  
CREATE ROLE replication WITH REPLICATION PASSWORD 'password' LOGIN;

  2. /etc/postgresql/9.5/main/postgresql.conf 파일 수정

    wal_level = hot_standby
max_wal_senders = 2
max_replication_slots = 2

  3. /etc/postgresql/9.5/main/hba.conf 파일 수정

    아래 문구 추가

    host  all   all   192.168.0.138/24  trust
host  replication   replication   192.168.0.138/24  md5
host  all   all   ::1/128    trust

  4. postgresql 재시작

STANDBY SERVER 세팅

  1. /etc/postgresql/9.5/main/pg_hba.conf 파일 수정 아래문구 추가

    host  all  all 192.168.0.0/24 trust

  2. postgres 유저 추가

    # psql을 Postgres 유저로 접속 후 유저 생성

sudo -u postgres psql  
CREATE ROLE replication WITH REPLICATION PASSWORD 'password' LOGIN;

  3. master server backup 진행

    # 백업 진행하기 전 postgresql 종료
systemctl stop postgresql

# 백업은 꼭 postgres 유저로 진행해야 됨 (중요)
su postgres

# /var/lib/postgresql/9.5/main 아래의 데이터는 데이터 베이스가 동작하는 main 디렉토리
# 하위 데이터가 없어야 백업이 정상적으로 실행됨
rm -rf /var/lib/postgresql/9.5/main/*
pg_basebackup -h 192.168.0.123 -D /var/lib/postgresql/9.5/main/ -U replication -P -v -X stream

  4. /var/lib/postgresql/9.5/main/recovery 파일 생성

    standby_mode='on'
primary_conninfo='host=192.168.0.123 port=5432 user=replication password=password'
primary_slot_name='repl_slot_01'
trigger_file='/var/lib/postgresql/9.5/main/failover_trigger'

  5. postgresql 실행

    systemctl start postgresql

replication 실행 확인

  1. master 서버에서의 확인

    select * from pg_stat_replication;

    아래처럼 1 row 가 출력되면 1개의 replication이 돌고 있는 것

  2. slave 에서의 확인

    /var/log/postgresql/postgresql-9.5-main.log 로그에서 아래와 같이 recovery 로그가 있으면 성공

  3. master / slave 모두 확인

    ps -ef | grep post 를 입력했을때 아래와 같이 recovering 이라는 프로세스가 돌고있으면 성공!

혹시 틀린부분 있으면 피드백 부탁드립니다!
